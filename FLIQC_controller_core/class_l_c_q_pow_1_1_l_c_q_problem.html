Run solver passing the desired LCQP in (file) dense format (qpOASES is used on subsolver level). All matrices are assumed to be stored row-wise.

Parameters
Q_fileThe objective's hessian matrix.
g_fileThe obective's linear term.
L_fileThe matrix selecting the left hand side of the complementarity pairs.
R_fileThe matrix selecting the right hand side of the complementarity pairs.
lbL_fileThe lower bounds associated to the complementarity matrix _L. A NULL leads to zero bounds.
ubL_fileThe upper bounds associated to the complementarity matrix _L. A NULL pointer can be passed if no upper bounds exist.
lbR_fileThe lower bounds associated to the complementarity matrix _R. A NULL leads to zero boudns.
ubR_fileThe upper bounds associated to the complementarity matrix _R. A NULL pointer can be passed if no upper bounds exist.
A_fileThe constraint matrix. A NULL pointer can be passed if no linear constraints exist.
lbA_fileThe lower bounds associated to the constraint matrix _A. A NULL pointer can be passed if no lower bounds exist.
ubA_fileThe upper bounds associated to the constraint matrix _A. A NULL pointer can be passed if no upper bounds exist.
lb_fileThe box constraint's lower bounds. A NULL pointer can be passed if no lower bounds exist.
ub_fileThe box constraint's upper bounds. A NULL pointer can be passed if no upper bounds exist.
x0_fileThe initial guess for the optimal primal solution vector. If a NULL pointer is passed the zero vector is used.
y0_fileThe initial guess for the optimal dual solution vector. If a NULL pointer is passed, then the initialization depends on the subsolver and its options.
Returns
SUCCESSFUL_RETURN if a solution is found. Otherwise the return value will indicate an occured error.
